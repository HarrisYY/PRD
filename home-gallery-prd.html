PRD：首页画廊组件展示框架调整

将网站首页画廊组件调整为底部模块展示，默认展示 8 款内容，采用一行 2 个瀑布流布局，并通过 View More / Load More 向下展开查看更多内容。

所属模块：首页 / Gallery 上线时间：2026-06-08 优先级：P1 交互逻辑：除展示框架外保持不变
需求概述 项目背景 当前逻辑 需求逻辑 原型设计 流程说明 测试用例 风险评估

需求项目标规则说明

2. 项目背景

当前首页画廊组件需要优化信息呈现密度与用户浏览路径。业务侧希望在首页底部增加更明确的内容承接区域，让用户在完成首页主要信息浏览后，可以继续查看更多画廊内容，从而提升内容曝光、点击深度与页面停留时间。

本次调整参考手绘框架：外层为首页底部模块容器，内部默认展示 4 组双列内容卡片，即 8 款内容，下方提供居中的 View More 入口。用户点击后继续向下展开更多内容。

用户价值降低用户寻找更多内容的成本，默认露出更多画廊内容。
业务价值提升首页底部区域利用率、画廊内容曝光与点击转化。
研发价值在保留既有交互和数据逻辑基础上完成展示层调整，控制改动范围。

4. 需求逻辑说明

4.1 核心功能

功能点需求描述验收标准
首页底部展示画廊组件放置在首页底部，位于 Footer 之前。访问首页时，用户向下浏览可在底部看到完整画廊模块。
默认展示 8 款模块初始化时最多展示 8 个画廊内容项。数据量大于等于 8 时展示 8 个；少于 8 时展示实际数量。
双列瀑布流内容卡片采用 2 列瀑布流布局，图片高度可不同。桌面端和移动端均保持一行 2 个内容项，卡片间距一致，不重叠。
查看更多入口默认内容下方展示 View More / Load More 按钮。当存在超过 8 个内容项时展示按钮；无更多内容时隐藏按钮。
向下展开点击 View More 后，在当前模块下方继续加载更多内容。新内容追加到原列表下方，不跳页、不打断当前页面浏览状态。

4.2 非核心功能

  • 模块标题、按钮文案可沿用现有多语言配置；英文环境建议使用 View More，如当前站点已使用 Load More，则保持现有文案。
  • 内容卡片标题、标签、价格、描述等字段展示规则保持现有配置，不因本次需求新增字段。
  • 图片懒加载、占位图、加载失败兜底图沿用现有全站或组件策略。

4.3 不做的功能

  • 不新增画廊内容详情页。
  • 不调整画廊内容的数据筛选、排序、推荐算法。
  • 不新增收藏、分享、评论等交互。
  • 不改变首页其他模块的位置、样式和业务逻辑。
  • 不改变 Footer 展示和站点导航结构。

4.4 规则细节

场景规则
数据量为 0画廊模块不展示，避免首页底部出现空白容器。
数据量 1-7展示实际数量，不展示 View More / Load More 按钮。
数据量等于 8展示 8 个内容项，不展示 View More / Load More 按钮。
数据量大于 8默认展示前 8 个内容项，并展示 View More / Load More 按钮。
点击查看更多按现有分页或加载更多规则继续追加内容；若加载完成后无更多内容，则隐藏或置灰按钮。
加载失败保留当前已展示内容，按钮恢复可点击状态，并按现有错误提示策略提示用户。

6. 主流程、分支流程、异常流程

6.1 主流程

1. 用户进入网站首页。
2. 页面按现有逻辑加载首页各模块与画廊组件数据。
3. 用户向下浏览至首页底部，在 Footer 前看到画廊组件。
4. 画廊组件默认展示前 8 个内容项，布局为一行 2 个瀑布流。
5. 若存在更多内容，用户点击 View More / Load More。
6. 系统在当前列表下方追加更多画廊内容，其他交互逻辑保持不变。

6.2 分支流程

分支处理逻辑
内容不足 8 个展示实际内容数量，不补空卡片，不展示查看更多按钮。
无更多内容默认展示内容后不出现查看更多按钮，或加载到最后一页后隐藏按钮。
多语言环境按钮文案按现有站点多语言配置展示，英文优先使用 View More。
点击内容卡片按现有逻辑跳转到详情页、专题页或目标链接。

6.3 异常流程

异常场景系统处理用户感知
画廊接口失败不渲染空模块或按现有组件兜底策略展示。首页其他模块正常可用。
加载更多失败保留已展示内容，按钮恢复可点击，保留原有错误提示策略。用户可重新点击加载。
图片加载失败使用现有默认占位图或错误图。卡片结构不塌陷，仍可点击。
接口返回重复数据按现有数据去重或展示策略处理，本需求不新增去重规则。与当前线上行为一致。

8. 风险评估

8.1 依赖项

依赖对象依赖内容完成要求
前端研发调整首页底部挂载位置、默认数量、双列瀑布流样式、查看更多状态。6 月 8 日前完成开发、联调、验收。
后端 / CMS确认画廊数据源可支持超过 8 条内容与分页加载。若现有接口已支持，则无需新增接口。
设计确认模块间距、按钮样式、卡片边距与视觉细节。按现有首页视觉规范落地。
数据 / 埋点校验曝光、点击、加载更多事件。上线前完成关键事件验收。

8.2 风险点与预案

风险点影响应对预案
瀑布流高度计算异常可能出现布局错位、重叠或大面积空白。优先使用成熟 CSS columns 或现有 Masonry 组件；覆盖多图高差测试。
首页底部加载内容过多影响页面性能和首屏之后滚动体验。默认仅展示 8 个，更多内容按现有分页懒加载策略追加。
移动端一行 2 个过窄可能影响图片和文案可读性。移动端保留 2 列但压缩文案展示，图片优先，文案超出截断。
按钮文案不一致不同语言或页面出现 View More / Load More 混用。上线前确认当前站点语言包；若已有统一文案，沿用现有配置。
上线时间紧6 月 8 日上线窗口较近，联调和回归时间有限。严格控制范围，仅调整展示框架；不引入新接口、新字段和新复杂交互。
上线验收清单
  • 首页底部 Footer 前展示画廊组件。
  • 默认展示 8 款内容。
  • 布局为一行 2 个瀑布流。
  • 存在更多内容时展示 View More / Load More。
  • 点击后向下追加更多内容。
  • 原有卡片点击、跳转、曝光、加载更多埋点保持可用。
  • 桌面端、平板端、移动端展示正常。
  • 无数据、少于 8 条、等于 8 条、大于 8 条均符合规则。
